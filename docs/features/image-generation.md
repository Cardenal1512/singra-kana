# Image Generation Rules

## General Style
- The visual style must be simple, cute and child-friendly.
- Avoid realism or semi-realistic rendering.
- Prefer soft rounded shapes and clean silhouettes.
- Images should feel calm, playful and easy to read on mobile devices.
- Avoid excessive visual noise or overly complex compositions.

## Format
- All generated assets must be exactly 512x512.
- Final app assets must be stored as transparent WEBP.
- If the image provider returns PNG, convert it to WEBP before storing.
- No background should be included unless explicitly requested.

## Composition
- The main subject must remain clearly visible at small mobile sizes.
- Avoid cluttered scenes or excessive detail.
- Keep compositions centered and balanced.
- Avoid tiny decorative elements that become unreadable on mobile.

## Text
- Never include text inside the image.
- Never include logos, labels or watermarks.

## Colors
- Prefer soft pastel palettes and non-aggressive colors.
- Avoid overly saturated or neon colors.
- Singra’s canonical colors are the only exception and must remain consistent:
  - Orange hoodie
  - Yellow body
  - Black beanie
  - Red beanie symbol

## Backgrounds
- Transparent background by default.
- If a background is explicitly requested, it should remain minimal, soft and stylized.

## Character Consistency
- Respect the canonical Singra design.
- Spiral glasses are mandatory.
- Keep the kawaii proportions and expressive face.
- Maintain consistency across all generated assets.
