# UI Rules

## General Philosophy
- UI should feel calm, playful and clean.
- Avoid cluttered layouts.
- Avoid excessive nesting of cards.

## Layout
- Minimize scrolling whenever possible.
- Main actions should remain visible.
- Back button should stay near the top.
- Bottom actions should stay near the bottom.

## Mobile First
- Prioritize iPhone experience above all.
- UI must remain readable on small screens.

## Animations
- Animations should feel soft and cozy.
- Avoid aggressive or overly fast transitions.
- Prefer subtle fades and smooth movement.

## Components
- Prefer reusable shared components.
- Avoid duplicating button styles or layout logic.
